Token-Efficient Multi-Model Handoff

Store compact, content-addressed handoffs once, then reuse them across models without re-sending context.

Scenario

You start with Model A to analyze and summarize. You want Model B to continue with the exact same context, without re-explaining the entire codebase or re-uploading files.

Workflow

1. Pack the handoff 
cargo run -- pack \
  --summary "API surface stabilized, next: implement X" \
  --file src/lib.rs \
  --cdom
2. Share the CID 
# copy the node CID from output
node: <node-cid>
3. Rehydrate anywhere 
cargo run -- show <node-cid>
cargo run -- unpack <node-cid> --out-dir ./handoff

What You Save

You avoid re-sending large context across models. The handoff packet and optional CDOM bundle provide a compact summary with direct links to the exact artifacts.

catbus stores handoffs in ket’s content-addressed store, preserving provenance in the DAG and enabling reliable multi-model workflows.
